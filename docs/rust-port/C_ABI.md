# `snapdir-ffi` C ABI Reference

> **Audience:** Authors of the Go (CGo), C/C++, Zig, and Java (JNA) bindings
> (Phases 39–42), and any downstream C consumer.  
> **Crate:** `crates/snapdir-ffi`  
> **Header:** `include/snapdir.h` (cbindgen-generated, C99 — **do not hand-edit**)  
> **ABI version:** `SNAPDIR_ABI_VERSION 1`  
> **Version parity:** Wraps the frozen `snapdir-api` 1.10.0 facade.

---

## 1. Overview

`snapdir-ffi` is a **C ABI wrapper** around the `snapdir-api` facade crate.
It ships as two binary artifacts:

| Artifact | Use case |
|---|---|
| `libsnapdir_ffi.so` / `libsnapdir_ffi.dylib` | cdylib — JNA, Zig `@cImport`, `dlopen` |
| `libsnapdir_ffi.a` | staticlib — CGo static linking |

The public C interface is declared in **`include/snapdir.h`**, generated by
`cbindgen` from the Rust sources.  The file is C99-compatible and includes
`<stdint.h>` and `<stdbool.h>`.  Never edit the header directly; re-generate
it with `cargo build -p snapdir-ffi` (the `build.rs` script runs cbindgen).

The **ABI version** is exposed as a preprocessor constant:

```c
#define SNAPDIR_ABI_VERSION 1
```

`SNAPDIR_ABI_VERSION` starts at `1` and is bumped **only** on a breaking
C-ABI change.  It is deliberately decoupled from the crate semantic version:
the crate semver may advance without bumping the ABI version.

The `snapdir_version()` function returns the snapdir library version string
(e.g. `"1.10.0"`); see §3.1.

## 2. Memory contract

Every `extern "C"` function in `snapdir-ffi` obeys the following invariants
without exception.  Binding authors must internalize this section before
calling any function.

### 2.1 Returned strings — `char *`

- Returned strings are **heap-allocated** by the library via
  `CString::into_raw()`.
- The caller **must** free them exactly once using
  **`snapdir_string_free(ptr)`**.
- `snapdir_string_free(NULL)` is a **safe no-op**.
- **Never** pass a returned string to the system `free()`, C++ `delete`, or
  any language-runtime deallocator — the allocator is Rust's global allocator
  and only `snapdir_string_free` is the correct deallocation path.
- All returned strings are valid **UTF-8**, NUL-terminated.

### 2.2 The opaque `SnapdirError` type

- `SnapdirError` is an **opaque** heap-allocated struct.  Its fields are
  invisible to C callers; use only the inspect functions.
- The caller **must** free a non-NULL error pointer exactly once using
  **`snapdir_error_free(err)`**.
- `snapdir_error_free(NULL)` is a **safe no-op**.
- `snapdir_error_message(err)` and `snapdir_error_code(err)` return
  **borrowed** `const char *` pointers that are **valid only for the lifetime
  of `err`**.  Do **not** free them separately.  They are invalidated when
  `snapdir_error_free(err)` is called.
- Double-freeing an error is **undefined behavior** — the contract is "free
  exactly once."

### 2.3 Error out-parameter — `SnapdirError **err_out`

All functions report errors through an **out-parameter** rather than the
return value:

```c
SnapdirError *err = NULL;
int rc = snapdir_fetch_blocking(id, store, 0, &err);
if (rc != 0) {
    fprintf(stderr, "[%s] %s\n",
            snapdir_error_code(err),
            snapdir_error_message(err));
    snapdir_error_free(err);
}
```

Rules:

| Condition | Return value | `*err_out` |
|---|---|---|
| Success | `0` (int fns) or non-NULL string (string fns) | `NULL` (unchanged) |
| Failure | `-1` (int fns) or `NULL` (string fns) | non-NULL `SnapdirError *` |

- Passing `NULL` for `err_out` is legal — error details are silently dropped
  (the function still returns `-1` or `NULL` to signal failure).
- On success, `*err_out` is **never written**; it retains whatever value the
  caller left there (initialize to `NULL`).

### 2.4 Static strings — `snapdir_version()`

`snapdir_version()` returns a pointer with **static lifetime** (valid for the
entire process lifetime).  The caller **must NOT free it**.  This is the only
`snapdir-ffi` function that returns an unowned pointer.

### 2.5 Strings-in — `const char *` parameters

- All input string parameters are expected to be valid **UTF-8**,
  **NUL-terminated** C strings.
- The FFI **copies** the string data; it never takes ownership of caller
  memory.
- `NULL` is allowed where a parameter is documented as **"unset"** or
  **optional**.  Required parameters (e.g. `path`, `id`, `store_uri`) must
  not be `NULL`; passing `NULL` for a required parameter produces a failure
  with an `IO_ERROR` error.

### 2.6 Panics never cross the boundary

Every entry point wraps its body in `std::panic::catch_unwind`.  A caught
Rust panic is converted into a `SnapdirError` with code `"INTERNAL"` and
written into `*err_out`.  The process is **never aborted** by a panic inside
a `snapdir-ffi` function (except inside `snapdir_init()` where a runtime
build failure aborts by design — an extremely rare condition).

### 2.7 Thread safety

All functions are safe to call from any thread once `snapdir_init()` has run
(or lazily on the first call to any function, since the runtime initialises
itself on demand).

## 3. Init and runtime

### 3.1 Function: `snapdir_init`

```c
void snapdir_init(void);
```

Initialises the **embedded shared multi-thread tokio runtime**.

- **Idempotent.** Safe to call any number of times and concurrently from
  multiple threads.  The runtime is created exactly once.
- **Lazy alternative.** All other `snapdir_*` functions lazily initialise the
  runtime on first use; `snapdir_init()` is optional but lets a binding
  front-load the initialisation cost at process startup.
- **Cannot fail visibly.** If the runtime cannot be built (extremely rare),
  the process aborts with a clear message.

### 3.2 Function: `snapdir_version`

```c
const char *snapdir_version(void);
```

Returns the snapdir library version string (e.g. `"1.10.0"`).

- The returned pointer has **static lifetime** — valid for the process
  lifetime.
- **Do NOT free** the returned pointer.

## 4. Function reference

Functions are grouped as they appear in `include/snapdir.h`.

### 4.1 Error inspection and cleanup

#### `snapdir_error_message`

```c
const char *snapdir_error_message(const SnapdirError *err);
```

Returns the human-readable error message for `err`.

- @param err A non-freed `SnapdirError` pointer, or `NULL`.
- @return Borrowed `const char *` valid within the lifetime of `err`.
  Returns `NULL` if `err` is `NULL`.

#### `snapdir_error_code`

```c
const char *snapdir_error_code(const SnapdirError *err);
```

Returns the stable error code string for `err`.  The code is one of the
8 stable values listed in §5, or `"INTERNAL"` for unexpected failures caught
by the `catch_unwind` boundary.

- @param err A non-freed `SnapdirError` pointer, or `NULL`.
- @return Borrowed `const char *` valid within the lifetime of `err`.
  Returns `NULL` if `err` is `NULL`.

#### `snapdir_error_free`

```c
void snapdir_error_free(SnapdirError *err);
```

Frees a `SnapdirError` previously returned via an error out-parameter.

- @param err Pointer previously written by a `snapdir-ffi` function, or
  `NULL` (no-op).
- Must not be called more than once on the same pointer.

#### `snapdir_string_free`

```c
void snapdir_string_free(char *s);
```

Frees a string previously returned by any `snapdir_*` function.

- @param s Pointer previously returned by a `snapdir-ffi` function, or
  `NULL` (no-op).
- Must not be called more than once on the same pointer.
- Must not be used on `snapdir_version()`'s return value (static lifetime).

### 4.2 Snapshotting — synchronous

These functions operate entirely within the calling thread (no async I/O).

#### `snapdir_manifest`

```c
char *snapdir_manifest(
    const char *path,
    const char *exclude,
    uint32_t    walk_jobs,
    bool        absolute,
    bool        no_follow,
    const char *checksum_bin,
    const char *cache_dir,
    const char *catalog,
    SnapdirError **err_out);
```

Walks `path` and returns the manifest text as an owned C string.

- @param path Directory to walk. Required; must not be `NULL`.
- @param exclude Extended-regex exclusion pattern applied to file paths.
  `NULL` = no exclusion.
- @param walk_jobs Parallel BLAKE3 hashing worker count. `0` = auto
  (CPU-count default).
- @param absolute If `true`, emit absolute paths instead of `./`-relative
  paths.
- @param no_follow If `true`, do not follow symbolic links.
- @param checksum_bin Checksum algorithm selector.  `NULL` or any
  unrecognised string selects the default `"b3sum"` (BLAKE3).  Pass
  `"md5sum"` or `"sha256sum"` to select those algorithms.
- @param cache_dir Override the local object-cache directory. `NULL` =
  default location.
- @param catalog Catalog adapter selection.  `NULL` = default adapter.
  `"none"` = suppress catalog recording.  Any other string selects a named
  adapter.
- @param err_out Error out-parameter. May be `NULL` to discard error details.
- @return Freshly-allocated manifest text string on success; the caller frees
  with `snapdir_string_free`. Returns `NULL` on failure with `*err_out` set.

#### `snapdir_id`

```c
char *snapdir_id(
    const char *path,
    const char *exclude,
    uint32_t    walk_jobs,
    const char *cache_dir,
    SnapdirError **err_out);
```

Computes the snapshot id for the directory at `path`.

- @param path Directory to walk. Required; must not be `NULL`.
- @param exclude Extended-regex exclusion pattern. `NULL` = no exclusion.
- @param walk_jobs Parallel hashing worker count. `0` = auto.
- @param cache_dir Cache directory override. `NULL` = default.
- @param err_out Error out-parameter.
- @return Freshly-allocated 64-character lowercase hex BLAKE3 snapshot id on
  success; the caller frees with `snapdir_string_free`. Returns `NULL` on
  failure with `*err_out` set.

#### `snapdir_id_from_manifest_text`

```c
char *snapdir_id_from_manifest_text(
    const char *manifest_text,
    SnapdirError **err_out);
```

Computes the snapshot id from an already-computed manifest text string
(e.g. a value previously returned by `snapdir_manifest`).

The returned id is the same 64-char lowercase-hex BLAKE3 id that
`snapdir_id` would produce for the original directory.

- @param manifest_text Valid NUL-terminated UTF-8 manifest text. Required;
  must not be `NULL`.
- @param err_out Error out-parameter.
- @return Freshly-allocated 64-char hex id on success; the caller frees with
  `snapdir_string_free`. Returns `NULL` on failure with `*err_out` set.

#### `snapdir_stage`

```c
char *snapdir_stage(
    const char *path,
    bool        keep,
    const char *cache_dir,
    SnapdirError **err_out);
```

Stages the directory at `path` in the local cache and returns the snapshot
id.

- @param path Directory to stage. Required; must not be `NULL`.
- @param keep If `true`, the source tree is preserved as-is after staging.
  If `false`, only the cache copy is kept — the source directory is still
  left intact by the FFI (the `keep` flag primarily controls CLI behavior).
- @param cache_dir Cache directory override. `NULL` = default.
- @param err_out Error out-parameter.
- @return Freshly-allocated 64-char hex snapshot id on success; the caller
  frees with `snapdir_string_free`. Returns `NULL` on failure with
  `*err_out` set.

### 4.3 Distribution — blocking

These functions perform network or filesystem I/O and block until completion.
Internally they route through the shared tokio runtime via `block_on`.  Only
blocking variants are provided in M1; async-callback variants are deferred.

#### `snapdir_push_blocking`

```c
char *snapdir_push_blocking(
    const char *source_path,
    const char *source_id,
    const char *store_uri,
    uint32_t    jobs,
    const char *limit_rate,
    uint32_t    max_retries,
    const char *cache_dir,
    SnapdirError **err_out);
```

Pushes a snapshot to `store_uri` and returns the snapshot id string.

Exactly one of `source_path` or `source_id` must be non-NULL:

- `source_path` — filesystem path to the directory to push.
- `source_id` — 64-hex id of a previously-staged snapshot (use when the
  snapshot has already been staged via `snapdir_stage`).

- @param source_path Directory path to push, or `NULL` when using
  `source_id`.
- @param source_id Previously-staged 64-hex snapshot id, or `NULL` when
  using `source_path`.
- @param store_uri Destination store URI (e.g. `"file:///tmp/store"`).
  Required; must not be `NULL`.
- @param jobs Max concurrent object transfers. `0` = default.
- @param limit_rate Bandwidth cap string (e.g. `"10M"` for 10 MB/s).
  `NULL` = unlimited.
- @param max_retries Max retry attempts per object. `0` = default (5).
- @param cache_dir Local cache directory override. `NULL` = default.
- @param err_out Error out-parameter.
- @return Freshly-allocated 64-char hex snapshot id on success; the caller
  frees with `snapdir_string_free`. Returns `NULL` on failure with
  `*err_out` set.

#### `snapdir_fetch_blocking`

```c
int snapdir_fetch_blocking(
    const char *id,
    const char *store_uri,
    uint32_t    jobs,
    SnapdirError **err_out);
```

Fetches a snapshot from `store_uri` into the local cache.

- @param id 64-hex snapshot id. Required.
- @param store_uri Source store URI. Required.
- @param jobs Max concurrent transfers. `0` = default.
- @param err_out Error out-parameter.
- @return `0` on success, `-1` on failure with `*err_out` set.

#### `snapdir_pull_blocking`

```c
int snapdir_pull_blocking(
    const char *id,
    const char *store_uri,
    const char *dest_path,
    bool        delete_extra,
    uint32_t    jobs,
    SnapdirError **err_out);
```

Pulls a snapshot from `store_uri` into `dest_path`, fetching objects and
materializing files in a single step.

- @param id 64-hex snapshot id. Required.
- @param store_uri Source store URI. Required.
- @param dest_path Filesystem path to materialize into. Required.
- @param delete_extra If `true`, delete destination files absent from the
  snapshot.
- @param jobs Max concurrent transfers. `0` = default.
- @param err_out Error out-parameter.
- @return `0` on success, `-1` on failure with `*err_out` set.

#### `snapdir_checkout_blocking`

```c
int snapdir_checkout_blocking(
    const char *id,
    const char *dest_path,
    bool        linked,
    bool        delete_extra,
    SnapdirError **err_out);
```

Checks out a snapshot from the **local cache** into `dest_path`.

The snapshot must already be present in the local cache (fetched via
`snapdir_fetch_blocking` or `snapdir_push_blocking`) before this call.

- @param id 64-hex snapshot id. Required.
- @param dest_path Filesystem path to materialize into. Required.
- @param linked If `true`, materialize using symlinks instead of copies.
- @param delete_extra If `true`, delete destination files absent from the
  snapshot.
- @param err_out Error out-parameter.
- @return `0` on success, `-1` on failure with `*err_out` set.

#### `snapdir_sync_blocking`

```c
int snapdir_sync_blocking(
    const char *id,
    const char *src_uri,
    const char *dst_uri,
    uint32_t    jobs,
    SnapdirError **err_out);
```

Syncs a snapshot from `src_uri` to `dst_uri`, copying only objects not
already present at the destination.

- @param id 64-hex snapshot id. Required.
- @param src_uri Source store URI. Required.
- @param dst_uri Destination store URI. Required.
- @param jobs Max concurrent transfers. `0` = default.
- @param err_out Error out-parameter.
- @return `0` on success, `-1` on failure with `*err_out` set.

#### `snapdir_verify_blocking`

```c
int snapdir_verify_blocking(
    const char *id,
    const char *store_uri,
    bool        purge,
    SnapdirError **err_out);
```

Verifies a snapshot in `store_uri` — checks that all manifest objects are
present and hash-consistent.

- @param id 64-hex snapshot id. Required.
- @param store_uri Store URI to verify against. Required.
- @param purge If `true`, remove corrupt objects from the local cache during
  verification.
- @param err_out Error out-parameter.
- @return `0` if the snapshot is healthy, `-1` on failure (hash mismatch,
  missing objects, or I/O error) with `*err_out` set.

### 4.4 JSON multi-value results

These functions return serialized JSON arrays.  Bindings are expected to
parse the JSON natively (returning opaque iterators across C would add
unnecessary complexity).  All returned strings must be freed with
`snapdir_string_free`.

#### `snapdir_diff_json`

```c
char *snapdir_diff_json(
    const char **from_uris,
    const char **to_uris,
    const char  *id,
    bool         include_unchanged,
    const char  *on_conflict,
    SnapdirError **err_out);
```

Diffs two sets of stores and returns a JSON array of change entries.

- @param from_uris NULL-terminated array of source store URIs (each inner
  pointer must be non-NULL).
- @param to_uris NULL-terminated array of destination store URIs (same
  requirements).
- @param id Optional 64-hex snapshot id.  Currently informational — the API
  unions all manifests per side regardless.  Pass `NULL` for a straight A→B
  diff across distinct stores.
- @param include_unchanged If `true`, include unchanged entries (status `"="`)
  in the output.
- @param on_conflict Conflict policy when the same path appears in multiple
  manifests on one side: `"error"` (default, `NULL` maps here) or
  `"last-wins"`.
- @param err_out Error out-parameter.
- @return Freshly-allocated JSON string on success; `NULL` on failure.

**JSON shape:**

```json
[
  {"status": "A", "path": "./added.txt"},
  {"status": "D", "path": "./deleted.txt"},
  {"status": "M", "path": "./modified.txt"},
  {"status": "=", "path": "./unchanged.txt"}
]
```

Status glyphs:

| Glyph | Meaning |
|---|---|
| `"A"` | Added — present in `to`, absent in `from` |
| `"D"` | Deleted — present in `from`, absent in `to` |
| `"M"` | Modified — present in both, content differs |
| `"="` | Unchanged — present in both, content identical |

#### `snapdir_locations_json`

```c
char *snapdir_locations_json(
    const char *cache_dir,
    const char *catalog,
    SnapdirError **err_out);
```

Returns all catalog locations as a JSON array.

- @param cache_dir Cache directory override for legacy catalog path
  resolution. The actual catalog path is resolved from
  `$SNAPDIR_CATALOG_DB_PATH` or the default. `NULL` = default.
- @param catalog Catalog name selector (currently unused; pass `NULL`).
- @param err_out Error out-parameter.
- @return Freshly-allocated JSON string on success; `NULL` on failure.

**JSON shape:**

```json
[
  {"created_at": "2026-06-21T10:00:00Z", "id": "<64-hex>", "location": "file:///path/to/store"}
]
```

#### `snapdir_ancestors_json`

```c
char *snapdir_ancestors_json(
    const char *id,
    const char *catalog,
    SnapdirError **err_out);
```

Returns the ancestor chain for `id` as a JSON array (from the catalog).

- @param id 64-hex snapshot id. Required.
- @param catalog Catalog name selector (currently unused; pass `NULL`).
- @param err_out Error out-parameter.
- @return Freshly-allocated JSON string on success; `NULL` on failure.

**JSON shape:**

```json
[
  {"created_at": "2026-06-20T09:00:00Z", "id": "<64-hex>", "location": "file:///path/to/store"}
]
```

#### `snapdir_revisions_json`

```c
char *snapdir_revisions_json(
    const char *location,
    const char *catalog,
    SnapdirError **err_out);
```

Returns the revision history at `location` as a JSON array.

- @param location Store URI or catalog location path to look up. Required.
- @param catalog Catalog name selector (currently unused; pass `NULL`).
- @param err_out Error out-parameter.
- @return Freshly-allocated JSON string on success; `NULL` on failure.

**JSON shape:**

```json
[
  {
    "created_at": "2026-06-21T10:00:00Z",
    "id": "<64-hex>",
    "previous_id": "<64-hex-or-null>"
  }
]
```

`previous_id` is `null` for the initial revision.

### 4.5 Cache maintenance

#### `snapdir_verify_cache`

```c
int snapdir_verify_cache(
    const char *cache_dir,
    bool        purge,
    SnapdirError **err_out);
```

Verifies the local object cache at `cache_dir`.

- @param cache_dir The cache directory to verify. Required; must not be
  `NULL`.  A missing or empty cache directory is a successful no-op.
- @param purge If `true`, remove corrupt objects from the cache. The
  `purge` semantics are handled by the API layer.
- @param err_out Error out-parameter.
- @return `0` if the cache is clean (or empty), `-1` on failure with
  `*err_out` set.

#### `snapdir_flush_cache`

```c
int snapdir_flush_cache(
    const char *cache_dir,
    SnapdirError **err_out);
```

Flushes (empties) the local object cache at `cache_dir`.

- @param cache_dir The cache directory to flush. Required; must not be
  `NULL`.  A missing or empty cache directory is a successful no-op.
- @param err_out Error out-parameter.
- @return `0` on success, `-1` on failure with `*err_out` set.

## 5. Error codes

`snapdir_error_code` returns one of the following **stable string codes**.
Binding authors should `strcmp` against these values — not against the human-
readable message.

| Code | Meaning | Currently reachable |
|---|---|---|
| `IO_ERROR` | Filesystem or network I/O failure | Yes |
| `HASH_MISMATCH` | Object content does not match its BLAKE3 hash | Yes |
| `STORE_ERROR` | Store backend error (invalid URI, auth failure, etc.) | Yes |
| `IN_FLUX` | Source directory changed during manifest walk | Yes (detected by the walk) |
| `CATALOG_ERROR` | Catalog database error | Yes (when a catalog DB exists and is corrupt) |
| `INVALID_ID` | Malformed or unrecognised snapshot id | Yes |
| `INVALID_STORE` | Unrecognised or malformed store URI | Yes |
| `CONFLICT` | Conflicting manifests under `"error"` conflict policy | Yes (reachable via `on_conflict` in `snapdir_diff_json`) |

An additional non-stable code exists for internal failures:

| Code | Meaning |
|---|---|
| `INTERNAL` | Unexpected Rust panic caught at the FFI boundary |

`INTERNAL` is not one of the 8 stable public codes and should be treated as
an unrecoverable internal error.

## 6. JSON shapes (summary)

| Function | JSON type | Key fields |
|---|---|---|
| `snapdir_diff_json` | array of objects | `status` (A/D/M/=), `path` |
| `snapdir_locations_json` | array of objects | `created_at`, `id`, `location` |
| `snapdir_ancestors_json` | array of objects | `created_at`, `id`, `location` |
| `snapdir_revisions_json` | array of objects | `created_at`, `id`, `previous_id` |

All timestamps are ISO 8601 UTC strings.  All ids are 64-character lowercase
hex BLAKE3 hashes.  `previous_id` is a JSON `null` for the initial revision.

Bindings are expected to parse these JSON strings natively (e.g. `encoding/json`
in Go, `json` in Python, `nlohmann/json` in C++, `org.json` in Java).

## 7. Linking and pkg-config

`packaging/snapdir.pc` is a standard pkg-config metadata file:

```
pkg-config --cflags --libs snapdir
```

This emits the correct `-I` and `-L -lsnapdir_ffi` flags for the installation
prefix.

**Static linking** requires the private dependencies listed in
`Libs.private`:

```
pkg-config --static --libs snapdir
# Adds: -lpthread -ldl -lm
```

### Development / in-tree builds

During development the build artifacts live at:

```
target/<profile>/libsnapdir_ffi.{so,dylib,a}   (built by cargo)
include/snapdir.h                                (repo root, cbindgen-generated)
```

Override the prefix at query time:

```sh
PKG_CONFIG_PATH=packaging \
  pkg-config --define-variable=prefix="$PWD" snapdir
```

### Catalog environment variable

The catalog database path is controlled by the environment variable
`$SNAPDIR_CATALOG_DB_PATH`.  The `catalog` string parameters in the API
select the catalog **name** (adapter), not the DB path.  The default
catalog is a `redb`-backed SQLite-compatible database at the system-default
location unless `$SNAPDIR_CATALOG_DB_PATH` overrides it.

## 8. Minimal C example

The following example demonstrates the common push → pull round-trip.  It
compiles with `clang -std=c99` against the generated header and the cdylib.

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include "snapdir.h"

/* Utility: print and free an error, then abort. */
static void die_on_error(SnapdirError *err, const char *context) {
    if (err) {
        fprintf(stderr, "snapdir error in %s [%s]: %s\n",
                context,
                snapdir_error_code(err),
                snapdir_error_message(err));
        snapdir_error_free(err);
        exit(1);
    }
}

int main(void) {
    /* 1. (Optional) Front-load runtime initialisation. */
    snapdir_init();

    /* 2. Print the library version (static — do NOT free). */
    printf("snapdir %s (ABI %d)\n", snapdir_version(), SNAPDIR_ABI_VERSION);

    /* 3. Compute the manifest of a directory. */
    SnapdirError *err = NULL;
    char *manifest_text = snapdir_manifest(
        "/tmp/my-data",  /* path      */
        NULL,            /* exclude   */
        0,               /* walk_jobs — auto */
        false,           /* absolute  */
        false,           /* no_follow */
        NULL,            /* checksum_bin — b3sum (BLAKE3) */
        NULL,            /* cache_dir — default */
        "none",          /* catalog   — suppress recording */
        &err);
    die_on_error(err, "snapdir_manifest");

    /* 4. Derive the snapshot id from the manifest text. */
    char *snap_id = snapdir_id_from_manifest_text(manifest_text, &err);
    die_on_error(err, "snapdir_id_from_manifest_text");
    snapdir_string_free(manifest_text);

    printf("snapshot id: %s\n", snap_id);

    /* 5. Push to a local file:// store. */
    char *pushed_id = snapdir_push_blocking(
        "/tmp/my-data",       /* source_path */
        NULL,                 /* source_id   */
        "file:///tmp/store",  /* store_uri   */
        0,                    /* jobs — default */
        NULL,                 /* limit_rate  */
        0,                    /* max_retries — default (5) */
        NULL,                 /* cache_dir   */
        &err);
    die_on_error(err, "snapdir_push_blocking");
    printf("pushed id:   %s\n", pushed_id);
    snapdir_string_free(pushed_id);

    /* 6. Pull (fetch + checkout) from the same store to a new destination. */
    int rc = snapdir_pull_blocking(
        snap_id,              /* id          */
        "file:///tmp/store",  /* store_uri   */
        "/tmp/my-data-out",   /* dest_path   */
        false,                /* delete_extra */
        0,                    /* jobs        */
        &err);
    die_on_error(err, "snapdir_pull_blocking");

    snapdir_string_free(snap_id);

    printf("pull succeeded — data materialized at /tmp/my-data-out\n");
    return 0;
}
```
